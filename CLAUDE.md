# Hibi Kioku — Claude Code conventions

Web SRS review client for the Hibi ecosystem. Review due cards and browse the library from the desktop. Companion to the mobile mining apps (hibi-koe = listening, hibi-yomi = reading) which create cards; kioku is where you actually study them.

## Stack

- Vite 7 + React 19.2 + TypeScript 5.9 (strict, `noUncheckedIndexedAccess`, `verbatimModuleSyntax`)
- TanStack Router 1 (file-based, typed search params, codegen `routeTree.gen.ts`)
- TanStack Query 5 for server-state caching (review queue prefetch, optimistic submit)
- `hibi-client` SDK over a bearer API key. The key is pasted into Settings and stored in `localStorage` (web equivalent of koe's `expo-secure-store`)
- vitest 4 for unit tests (`src/**/*.test.ts(x)`)
- Biome 2.4 for lint/format, pnpm 10 for package management
- Deploys to Vercel as a static SPA. `vercel.json` rewrites all routes to `/index.html`

## Design system

- Foundations: **Torakaa Design System** (5 variants: paper, dark, sepia, high-contrast, custom). Default: `paper`. Implemented as CSS variables in `src/styles/index.css`, switched by setting `data-theme="…"` on the document root.
- Hibi extensions reference: `/home/ilios/dev/hibi/DESIGN_SYSTEM.md`. Components (SentenceCard, FuriganaText, ReviewButton, KanjiTable, Heatmap) are re-implemented here from spec — not copied from `apps/design`. Don't drift from the spec; if a behavior is unclear, read the design app's reference impl in `/home/ilios/dev/hibi/apps/design/src/components/extensions/`.
- Always reference theme tokens by CSS variable name (`var(--color-accent)`, `var(--space-s4)`), never raw values.
- Editorial vibe: hairline rules, sharp corners (radii ≈ 0), accent used sparingly, numbers/titles in Newsreader serif, labels in mono Geist Mono uppercase with wide tracking.

## Where things live

- **Auth gate** — `beforeLoad` on protected routes calls `getApiKey()`; if missing, throws `redirect({ to: "/settings" })`. Validation (one-shot `cards.list({ limit: 1 })`) runs in Settings on paste.
- **API client** — `getHibiClient()` in `lib/hibiClient.ts` returns a memoized `HibiClient` built from the stored key. `localStorage` is the source of truth; the function throws if missing.
- **Review queue** — TanStack Query `useDueCards()` hook in the Review route pre-fetches via the route loader. Submit uses `useMutation` with optimistic advance to the next card.
- **Library state** — URL-driven via TanStack Router's `validateSearch` (Zod). `q`, `tag`, `sort`, `cursor` round-trip through the URL so back/forward and shareable links work.
- **Theme** — `hydrateTheme()` runs once in `main.tsx` *before* rendering to avoid a flash of default. The user's selection is stored under `kioku-theme` in `localStorage`.

## Hibi API requirements

This app needs `hibi-client@^0.4.0` (search/sort + signed media URLs in card responses). During monorepo development, link via `file:../hibi/packages/api-client`; switch to a published `^0.4.0` before deploying to Vercel.

## Commit style

Short imperative subject, conventional commit prefix (chore/feat/fix/docs/refactor/test). Bullet body for the *why*; the diff explains the what. Many small, atomic commits — one logical change each. Never `--no-verify` or amend pushed commits.

## Out of scope (v0)

Card editing (PATCH/DELETE from UI), full PWA / offline review queue, dictionary popup / tokenized text interactivity in the library, Supabase login alternative to the API key, extracting `@hibi/ui` shared package. Add later; do not pre-plumb.
